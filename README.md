# Mystery Shopper Backend — Таємний гість

Реальний бекенд для форми оцінки «Таємний гість»: приймає відповіді форми та
файли фото/відео, зберігає відповіді в Postgres, а файли — на диску,
розкладені по папках `Країна/Місто/Адреса`.

Що всередині:

- `server/index.js` — Express-сервер: API + роздача форми (`public/index.html`)
- `server/db.js` — підключення до Postgres і створення таблиці `reviews`
- `public/index.html` — сама форма й дашборд статистики (та ж, що в
  Cowork-артефакті, але тепер під'єднана до реального API замість
  захардкодженого масиву)

## Крок 1. Створити репозиторій на GitHub

1. Зайдіть на [github.com/new](https://github.com/new).
2. Назва репозиторію — будь-яка, наприклад `mystery-shopper-backend`.
3. Залиште «Public» або «Private» на власний розсуд, **не** додавайте README/.gitignore
   на цьому кроці (вони вже є в проєкті).
4. Натисніть «Create repository» — GitHub покаже вам URL виду
   `https://github.com/<ваш-логін>/mystery-shopper-backend.git`.

## Крок 2. Завантажити код у репозиторій

У вас вже є готова папка проєкту (те, що я підготував). У терміналі, перебуваючи
в цій папці:

```bash
git init
git add .
git commit -m "Initial commit: mystery shopper backend"
git branch -M main
git remote add origin https://github.com/<ваш-логін>/mystery-shopper-backend.git
git push -u origin main
```

Якщо GitHub попросить авторизацію — увійдіть через браузер або
[особистий токен доступу](https://github.com/settings/tokens).

## Крок 3. Розгорнути на Railway

1. Зайдіть на [railway.app](https://railway.app) і увійдіть (можна через GitHub-акаунт).
2. **New Project → Deploy from GitHub repo** → оберіть щойно створений репозиторій.
3. Railway автоматично визначить, що це Node.js-проєкт (за `package.json`) і
   запустить `npm install` та `npm start`.

## Крок 4. Додати базу даних Postgres

1. У цьому ж Railway-проєкті натисніть **New → Database → Add PostgreSQL**.
2. Все — Railway сам створить змінну середовища `DATABASE_URL` і автоматично
   прокине її в ваш сервіс (сервер підхоплює її з `process.env.DATABASE_URL`
   в `server/db.js`, нічого руками прописувати не потрібно).
3. Таблиця `reviews` створюється автоматично при першому запуску сервера.

## Крок 5. Додати сховище для фото/відео (Volume)

Без цього кроку завантажені файли **зникнуть при наступному деплої** —
Railway перезбирає контейнер з нуля щоразу.

1. На вкладці сервісу (не бази даних, а саме вашого Node-застосунку) відкрийте
   **Settings → Volumes → New Volume**.
2. Вкажіть mount path, наприклад `/data/uploads`.
3. У **Variables** цього сервісу додайте змінну:
   ```
   UPLOAD_DIR=/data/uploads
   ```
4. Перезапустіть деплой (Railway зробить це сам після збереження змінної).

## Крок 6. Отримати публічну адресу

1. У сервісі відкрийте **Settings → Networking → Generate Domain**.
2. Railway видасть посилання типу `https://mystery-shopper-backend-production.up.railway.app`.
3. Відкрийте його в браузері — має завантажитись форма «Таємний гість».
   Це і є посилання, яке можна віддавати тайним гостям для заповнення.

## Як це працює

- Вкладка **«Форма оцінки»**: при натисканні «Надіслати» браузер відправляє
  `POST /api/reviews` (оцінки, коментарі, NPS тощо) і, якщо обрані файли —
  `POST /api/upload` (multipart, з полями `country`, `city`, `location`,
  `orderId`). Файли лягають у `UPLOAD_DIR/<Країна>/<Місто>/<Адреса>/`.
- Вкладка **«Статистика»**: при відкритті робить `GET /api/reviews` і будує
  графіки та таблицю з реальних даних бази — більше не з захардкодженого масиву.
- `GET /api/files?country=..&city=..&location=..` — список файлів у конкретній
  папці (ім'я, розмір, посилання на завантаження).

## Локальний запуск (необов'язково, для перевірки перед деплоєм)

```bash
npm install
cp .env.example .env
# пропишіть у .env свій DATABASE_URL (можна тимчасову базу на neon.tech або supabase.com)
node --env-file=.env server/index.js
```

Відкрийте http://localhost:3000

## Обмеження поточної версії

- Немає адмін-панелі для перегляду/видалення окремих перевірок — тільки
  дашборд статистики та `GET /api/reviews`.
- Немає авторизації — будь-хто з посиланням може і заповнити форму, і
  переглянути `/api/reviews`. Якщо форма піде за межі довіреної команди,
  варто додати просту перевірку (наприклад, секретний параметр у посиланні
  або базову автентифікацію на рівні Railway).
- Ліміт розміру файлу — 300 МБ (можна змінити в `server/index.js`,
  константа `limits.fileSize`).
